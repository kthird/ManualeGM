Includere file negli eseguibili

Questa funzionalità è disponibile solo nella versione Standard Edition of GameMaker.

Come è stato indicato precedentemente, puoi creare versioni indipendenti (eseguibili) del tuo gioco. A volte il tuo gioco utilizzerà file aggiuntivi. Per esempio potresti voler includere file video o di testo che vengano usati nel gioco. In alcuni casi vorrai aggiungere DLL o immagini e suoni che vengano caricati durante l'esecuzione del gioco. Potresti fornire questi elementi (separatamente) insieme all'eseguibile ma è a volte più comodo includerli nello stesso file. In questo modo è necessario distribuire un solo file. I file inclusi saranno per definizione esportati quando il gioco venga avviato.

Puoi indicare i file che desideri includere nell'eseguibile scegliendo Included Files (File Inclusi) dal menù Resources (Risorse). Ti verrà mostrata la seguente finestra:

Nella parte alta della finestra c'è una lista di file da includere. Usa Add (Aggiungi) per aggiungere un nuovo file alla lista, Change (Cambia) per cambiare un file selezionato, Delete (Cancella) per eliminarlo, e Clear (Pulisci) per cancellare l'intera lista. Quando aggiungi o cambi un elemento, ti apparirà la seguente finestra:

Qui puoi indicare alcune proprietà. Puoi selezionare il tasto Load (Carica) per indicare quale file debba essere incluso. Puoi cambiare il nome nel campo File Name (Nome del File) se desideri salvare il file con un nome differente da quello originale.

C'è un certo numero di opzioni che è possibile selezionare. Quando selezioni Store in the editable .gm81 file (Immagazzina nel file modificabile .gm81) il file incluso verrà aggiunto non soltanto all'eseguibile ma anche alla versione modificabile del gioco. Il vantaggio è che puoi ad esempio (e.g.) mandare la versione modificabile a qualcun'altro ed essere sicuro che anche il file sia presente lì (con esso). Inoltre se desideri puoi rimuovere il file originale una volta incluso o spostarlo altrove senza problemi. Lo svantaggio è che la versione modificabile del gioco diventa così più grande e impiega più tempo nel caricamento.

Selezionare l'opzione Free memory after export (Libera la memoria dopo l'esportazione) significa che dopo che il file sia stato esportato sarà rimosso dalla memoria. Se vuoi essere in grado di esportarlo nuovamente in seguito, dovrai deselezionare questa opzione. Se un file che venga esportato già esista normalmente non viene riscritto. Puoi cambiare questo comportamento selezionando Overwrite existing files (Sovrascrivi i file esistenti). Inoltre, quando il gioco venga completato i file non vengono normalmente rimossi (a meno chè essi non si trovino nella cartella temporanea che viene completamente rimossa). Puoi cambiare questo selezionando Remove at game end (Rimuovi alla fine/termine del gioco). Un'avvertenza. Se stai provando il tuo gioco, il percorso dei file (di lavoro) utilizzato dal gioco è il percorso dove si trovi il file .gm81. Se anche i tuoi file d'inclusione si trovano nello stesso percorso e tu scegli di rimuoverli al termine del gioco potresti perderli tutti insieme! Così è meglio non posizionare questi file con il file .gm81 ma (e.g.) in una sottocartella!

Infine puoi indicare in quale cartella i file debbano essere esportati. Qui sono disponibili quattro opzioni. Nel caso predefinito i file sono esportati nella stessa cartella dove si trovi salvato il gioco indipendente (l'eseguibile). Questo è anche il percorso di lavoro per il gioco. Così il gioco può utilizzare già (soltanto) i nomi dei file per accedervi (non è necessario inserire tutto il percorso dei file)(non sono necessari i percorsi dei file). Questo lavora funzionerà correttamente se il gioco indipendente si trovi ad esempio (e.g.) sul disco rigido (hard disk) ma non funzionerà correttamente se è posizionato su di un dispositivo di sola lettura (e.g.), ad esempio su di un CD.

La seconda possibilità è indicare quali file debbano essere esportati nella cartella temporanea che viene creata durante l'avvio del gioco. Se selezioni questa opzione avrai bisogno di fornire il percorso fino alla cartella temporanea quando utilizzi il nome del file durante il gioco. Questo percorso può essere ottenuto usando la variabile interna (built-in variable) temp_directory. Non dimenticare di aggiungere la barra rovesciata ("\") in questo caso. Così ad esempio (e.g.) per avviare la riproduzione di un file video dovresti inserire il seguente codice: 

{
  splash_show_video(temp_directory+'\movie.avi',true);
}

Ricorda che questa cartella temporanea viene rimossa una volta che il gioco sia terminato . Così non puoi conservare partite salvate o particolari informazioni in questo caso. Scegli questa opzione solamente quando vuoi che il gioco venga avviato da un CD o quando non vuoi scrivere alcun file al di fuori del tuo gioco.

La terza opzione è indicare tu stesso la cartella nella quale esportare il file. Se non dovesse già esistere verrebbe creata. Ad esempio (e.g.) potresti specificare C:\MyGame. (Nota che devi fornire un percorso completo e che non dovresti includere una barra rovesciata ("\") alla fine.) Questa opzione è consentita soltanto quando il gioco non venisse eseguito in modalità protetta.

Infine, puoi indicare di non esportare il file automaticamente. In questo caso potresti usare ad esempio (e.g.) la funzione export_include_file(fname) per esportarlo tu stesso quando richiesto .
