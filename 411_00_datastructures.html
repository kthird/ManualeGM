Strutture dati

Questa funzionalità è disponibile solo nella versione Standard Edition di GameMaker.

Nei giochi c'è spesso la necessità di memorizzare informazioni. Ad esempio può essere necessario memorizzare elenchi di elementi che un giocatore conserva oppure memorizzare luoghi che devono ancora essere visitati. Si possono utilizzare le matrici per fare questo. Ma se si vogliono fare operazioni più complesse, come l'ordinamento dei dati o la ricerca di un elemento particolare, è necessario scrivere lunghi codici in GML che possono rallentare l'esecuzione.

Per rimediare a questo, GameMaker ha numerose strutture dati precostruite che possono essere accessibili tramite funzioni. Al momento ci sono sei diversi tipi di strutture dati disponibili: pile, code, liste, mappe, code di priorità, e griglie. Ciascuna di queste strutture dati è specializzata per fini diversi (vedi sotto).

Tutte le strutture dati funzionano globalmente nello stesso modo. È possibile creare una struttura dati con una funzione che restituisce un ID per essa. È possibile utilizzare questo ID per eseguire operazioni sulle strutture dati. Una volta terminate le operazioni occorre distruggere la struttura dati per liberare memoria. È possibile utilizzare più strutture nello stesso momento. Tutte le strutture sono in grado di memorizzare stringhe e valori reali.

Si noti che le strutture dati e il loro contenuto non vengono salvate quando si salva il gioco utilizzando le apposite funzioni. Se si utilizzano strutture dati e si desidera salvarle è necessario creare un proprio meccanismo per fare questo.

Quando si confrontano valori, ad esempio quando si cerca in una mappa o si ordina una lista, GameMaker deve decidere se due valori sono uguali. Per le stringhe e valori interi questo è facile, ma per i numeri reali, a causa di errori di arrotondamento, numeri uguali possono facilmente diventare diseguali. Per esempio (5/3) * 3 non è uguale a 5. Per evitare questo, viene usato un numero di precisione: quando la differenza tra due numeri è minore di tale precisione, essi sono considerati uguali. La precisione predefinita è di 0.0000001. È possibile modificare questa precisione utilizzando la seguente funzione:

ds_set_precision(prec) Imposta la precisione che verrà utilizzata per confronti di numeri reali.

Questa precisione è utilizzata in tutte le strutture dati, ma non in altri confronti in GML!
