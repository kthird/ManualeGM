Settare una connessione

Per comunicare due computer necessitano di un protocollo di connessione. Come molti giochi, Game Maker offre quattro tipi differenti di connessioni; IPX, TCP/IP, Modem e Seriale. La connessione IPX (per essere precisi è un protocollo) lavora in un modo trasparente. Può essere usata per giocare con altre persone nella stessa area locale. Deve essere installata sul computer per essere usata (Se non funziona consulta la documentazione di Windows o vai nel pannello di controllo alla sezione Rete ed aggiungilo). TCP/IP è il protocollo usato in internet. Può essere usato per giocare ad un gioco usando una linea internet, sapendo solo il tuo indirizzo IP. In una connessione locale non è necessario che tu fornisca il tuo indirizzo. Una connessione di tipo modem è usata esclusivamente per chi possiede un modem. Tu devi dare alcune opzioni del modem (una stringa di inizializzazione ed un numero di telefono) per poterlo usare. Infine quando usi una linea seriale (una connessione diretta tra computers) devi dare un numero di porta. Ci sono quattro funzioni GML che permettono di inizializzare queste connessioni:

mplay_init_ipx() inizializza una connessione IPX.
mplay_init_tcpip(addr) inizializza una connessione TCP/IP. addr è una stringa che può contenere un indirizzo web o IP come "www.gioco.es" o "123.123.123.12" possibilmente seguiti da il numero di porta (che non è indispensabile) come ":12". Solo quando partecipi ad una sessione (vedi dopo) devi provvedere il tuo indirizzo. In un'area locale non c'è bisogno che tu fornisca un indirizzo, ma devi comunque chiamare la funzione.
mplay_init_modem(initstr,phonenr) inizializza una connessione modem. initstr sta per stringa di inizializzazione del modem (può essere vuota). telefon è una stringa che conterrà il numero da chiamare (es. "0453232242"). Solo quando partecipi ad una sessione devi fornire il numero di telefono.
mplay_init_serial(portno,baudrate,stopbits,parity,flow) inizializza una connessione seriale. portno è il numero di porta, da 1 a 4. baudrate è la velocità di trasmissione da usare (100-256K). stopbits è il numero di stopbits (0 = 1 bit, 1 = 1.5 bit, 2 = 2 bits). parity indica il numero di parità (0 =none, 1=dispari, 2=pari, 3=mark). E flow indica il tipo di controllo di flusso (0=none, 1=xon/xoff, 2=rts, 3=dtr, 4=rts e dtr). Restituisce se ha successo. Una tipica chiamata seriale è mplay_init_serial (1,57600,0,0,4). Dare 0 come primo parametro per aprire una finestra di dialogo per l'utente per modificare le impostazioni.

Il gioco dovrebbe chiamare una di queste funzioni una sola volta. Tutte le funzioni ritornano valori se hanno avuto successo. Possono non avere successo se il particolare protocollo non è installato o supportato dalla macchina. Per verificare se vi sia una connessione disponibile è possibile utilizzare la seguente funzione:

mplay_connect_status() restituisce lo stato della connessione corrente. 0 = nessun collegamento, 1 = connessione IPX, 2 = connessione TCP / IP, 3 = connessione modem, e 4 = collegamento seriale.

Per terminare la chiamata di connessione:

mplay_end() termina la connessione corrente.

Quando si utilizza una connessione TCP / IP bisognerebbe chiedere alla persona con cui si vuole giocare l’indirizzo ip del proprio computer. La seguente funzione ti aiuta per fare questo:

mplay_ipaddress() restituisce l'indirizzo IP della macchina (ad esempio, '123 .123.123.12 ') come una stringa. È possibile ad esempio far visualizzare questo numero da qualche parte sullo schermo. NB: questa funzione è lenta, non bisogna chiamarla per tutto il tempo del gioco.
