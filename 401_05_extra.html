Variabili in più

È possibile creare nuove variabili, assegnando loro un valore (non è necessario dichiararle prima). Se si usa semplicemente solo il nome di una variabile, la variabile verrà memorizzata con l'istanza dell'oggetto corrente. Quindi non ci si aspetti di poterla usare quando si programmano le azioni di un altro oggetto (e il suo valore sarà diverso anche in altre istanze dello stesso oggetto). È anche possibile impostare e leggere variabili in altri oggetti mettendo il nome dell'oggetto con un punto prima del nome della variabile.

Per creare variabili globali, che sono visibili a tutte le istanze degli oggetti, occorre precedere al nome della variabile la parola global con un punto. Si può scrivere, per esempio, così: 

{
  if (global.doit)
  {
    // do something
    global.doit = false;
  }
}

In alternativa, è possibile dichiarare le variabili come globali come segue: 

globalvar <varname1>,<varname2>,<varname3>, ... ;

Una volta che la dichiarazione è stata eseguita, la variabile verrà sempre considerata come globale, senza la necessità di mettere la parola global e un punto di fronte ad essa. Ha solo bisogno di essere dichiarata una sola volta in un pezzo di codice che verrà eseguito. Dopo di che in tutti gli altri luoghi la variabile sarà considerata come globale.

A volte si vogliono solo delle variabili all'interno del pezzo di codice corrente o di uno script. In questo modo si evita di sprecare memoria e si è certi che non vi saranno conflitti di denominazione. È anche più veloce rispetto all'utilizzo di variabili globali. Per raggiungere questo obiettivo è necessario dichiarare le variabili all'inizio del pezzo di codice utilizzando la parola chiave var, come segue: 

var <varname1>,<varname2>,<varname3>, ... ;

Per esempio, si può scrivere: 

{
  var xx,yy;
  xx = x+10;
  yy = y+10;
  instance_create(xx,yy,ball);
}
