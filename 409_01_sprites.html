Sprites

Sono disponibili le seguenti funzioni per cambiare le proprietà degli sprite:

sprite_set_offset(ind,xoff,yoff) Imposta l'offset dello sprite con l'indice dato.

Le seguenti funzioni possono essere utilizzate per creare nuovi sprite e per rimuoverli.

sprite_duplicate(ind) Crea una copia dello sprite con l'indice dato e restituisce l'indice del nuovo sprite. Quando c'è un errore, restituisce -1.
sprite_assign(ind,spr) Assegna lo sprite indicato allo sprite ind. Quindi questo crea una copia dello sprite. In questo modo puoi facilmente impostare uno sprite esistente a uno differente, per esempio uno nuovo.
sprite_merge(ind1,ind2) Unisce le immagini dallo sprite ind2 allo sprite ind1, aggiungendole alla fine. Se la dimensione non combacia gli sprite vengono ridimensionati. Lo sprite ind2 non verrà cancellato!
sprite_add(fname,imgnumb,removeback,smooth,xorig,yorig) Aggiunge l'immagine memorizzata nel file fname al set di risorse degli sprite. Possono essere trattati molti tipi di file differenti. Quando l'immagine non è una gif animata può essere uno strip contenente un certo numero di sottoimmagini una dopo l'altra. Usa imgnumb per indicare quante sono (1 per una sola immagine). Per le gif animate, questo argomento non viene usato; viene usato il numero di immagini nella gif invece. removeback indica se tutti i pixel con il colore dello sfondo (il pixel in alto a sinistra) devono essere resi trasparenti. smooth indica se smussare i lati. xorig e yorig indicano la posizione dell'origine nello sprite. La funzione restituisce l'indice del nuovo sprite che puoi usare in seguito per disegnarlo o assegnarlo alla variabile sprite_index di un'istanza. Quando avviene un'errore, restituisce -1.
sprite_replace(ind,fname,imgnumb,removeback,smooth,xorig,yorig) Lo stesso, ma in questo caso viene sostituito lo sprite con l'indice ind. La funzione restituisce se non ci sono stati errori.
sprite_add_sprite(fname) Aggiunge lo sprite memorizzato nel file fname al set di risorse. Il file dev'essere un file .gmspr che viene salvato nella finestra delle proprietà degli sprite di GameMaker. Visto che il file contiene tutte le impostazioni dello sprite, non sono necessari altri argomenti. La funzione restituisce l'indice del nuovo sprite che puoi in seguito usare per disegnarlo o per assegnarlo alla variabile sprite_index. Restituisce -1 in caso di errore.
sprite_replace_sprite(ind,fname) Lo stesso, ma viene sostituito lo sprite con indice ind. La funzione restituisce se non ci sono stati errori.
sprite_create_from_screen(x,y,w,h,removeback,smooth,xorig,yorig) Crea uno sprite copiando una parte dallo schermo. removeback indica se devono essere resi trasparenti i pixel col colore dello sfondo (pixel in alto a sinistra). smooth indica se smussare i bordi. Questo rende possibile creare tutti gli sprite che vuoi. Disegna l'immagine nello schermo usando le funzioni apposite e crea uno sprite da esso. (Se non fai ciò nell'evento Draw puoi anche farlo in modo che non sia visibile nello schermo senza aggiornarlo). Gli altri parametri sono simili a quelli sopra. La funzione restituisce l'indice del nuovo sprite. È richiesta una certa attenzione qui. Anche se parliamo di schermo, in realtà è soltanto la parte in cui si disegna. Il fatto che c'è una finestra nello schermo e che l'immagine potrebbe essere ridimensionata non è rilevante.
sprite_add_from_screen(ind,x,y,w,h,removeback,smooth) Aggiunge un'area dello schermo come prossima subimage allo sprite con l'indice ind.
sprite_create_from_surface(id,x,y,w,h,removeback,smooth,xorig,yorig) Crea uno sprite copiando l'area data dalla surface con l'id dato. removeback indica se rendere trasparenti i pixel dello sfondo (pixel in alto a sinistra). smooth indica se smussare i bordi. Questo rende possibile creare tutti gli sprite che vuoi. Disegna l'immagine nella surface usando le funzioni apposite e in seguito crea uno sprite da esso. La funzione restituisce l'indice del nuovo sprite. Nota che i valori alpha vengono mantenuti nello sprite.
sprite_add_from_surface(ind,id,x,y,w,h,removeback,smooth) Aggiunge un'area della surface con l'id indicato come prossima subimage dello sprite con indice ind.
sprite_delete(ind) Cancella lo sprite dalla memoria, liberando la memoria utilizzata.

Queste funzioni esistono per cambiare l'aspetto dello sprite.

sprite_set_alpha_from_sprite(ind,spr) Cambia i valori alpha (la trasparenza) nello sprite con l'indice ind usando i valori di intensità nello sprite spr. Questo non può essere annullato.

Infine, puoi cambiare la maschera per le collisioni con le seguenti funzioni. Per favore sappi che questo non funziona correttamente quando un'istanza con quello sprite è già presente nella room.

sprite_collision_mask(ind,sepmasks,bboxmode,bbleft,bbtop,bbright,bbbottom,kind,tolerance) Cambia la maschera delle collisioni per lo sprite con indice ind. sepmasks indica se ci devono essere delle maschere separate per ogni sottoimmagine. bboxmode è la modalità per la bounding box (0=automatica, 1=piena immagine, 2=definita dall'utente). bbleft, bbright, bbtop, bbbottom indicano la bounding box nel caso in cui sia stata scelta la modalità 2. Puoi usare 0 per tutte le altre modalità. kind è il tipo di maschera (0=precisa, 1=bounding box, 2=disco, 3=diamante). tolerance indica la tolleranza nella trasparenza. (0=nessuna tolleranza, 255=totale tolleranza).
